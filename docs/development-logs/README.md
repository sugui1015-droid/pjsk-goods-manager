# 开发日志目录

本目录用于保存 PJSK Goods Manager 的开发日志。日志属于项目交接和审计资料，必须由 Git 正常追踪和备份。

## 存放规则

- 所有开发日志统一保存在 `docs/development-logs/`。
- 禁止在项目根目录另建“更新日志”等重复目录。
- 每个实际修改批次（代码、配置、迁移、文档、页面或测试）完成时，都必须同步更新日志。

## 文件命名规则

- 同一天的小修改优先追加到当天汇总日志。
- 较大的独立功能可以新增专题日志，命名格式为 `YYYY-MM-DD-英文简短功能名.md`。
- 文件名应稳定、可读，避免后续重命名。

## 追加和专题日志规则

- 日志只能记录已经实际完成和实际验证的结果，不能把计划中的功能写成已经完成。
- 人工验收未通过或未完成的功能，必须明确写为待验收或存在问题。
- 每个批次建议记录修改时间、目的、问题原因、修改文件、接口或数据库变化、前端变化、测试、验证命令、数据库影响、真实业务数据影响、Git 提交号和后续验收事项。

## 禁止事项

- 禁止擅自删除、移动、重命名或覆盖历史日志。
- 禁止通过 `.gitignore` 忽略本目录。
- 提交代码时必须同时提交对应日志；只提交代码不提交日志是不完整的变更。

## 恢复误移动或误删除

1. 立即停止提交。
2. 执行 `git status --short`、`git diff --name-status` 和 `git diff --cached --name-status` 确认影响范围。
3. 如果删除尚未提交，先执行 `git restore --staged -- docs/development-logs/<file>` 取消暂存，再执行 `git restore --worktree -- docs/development-logs/<file>` 恢复文件。
4. 如果是误移动，把文件恢复到原路径；必要时先恢复原文件，再删除错误新增路径。
5. 确认状态中不再出现开发日志的 `D` 或 `R` 后，再继续提交。
