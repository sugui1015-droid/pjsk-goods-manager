# Excel 解析规则定稿（沿用旧版）

## 当前基准文件

- `【生日羽排】汇总-20251001.xlsx`
- `26感谢祭单领.xlsx`

这两个文件都已被旧版解析器识别为“矩阵汇总表”。

## 总体流程

1. 读取工作簿全部工作表。
2. 每个工作表先尝试“矩阵汇总表”解析。
3. 若矩阵解析失败，再尝试“明细表”解析。
4. 两种格式都不匹配时，整个工作表跳过，不报错入库。

## 明细表规则

- 只在前 30 行内寻找表头。
- 若一行中至少命中 2 个字段别名，就认为这是表头行。
- 识别字段：
  - `cn`
  - `item_name`
  - `role`
  - `batch`
  - `quantity`
  - `unit_price`
  - `amount`
  - `collected`
- 金额列会去掉逗号和人民币符号再转数值。
- `quantity` 为空或解析后为 `0` 时，旧版会强制按 `1` 处理。
- `amount <= 0` 时，按 `quantity * unit_price` 回填。
- 只要 `cn / item_name / role / batch` 四列里至少有一列非空，该行就保留。

## 矩阵汇总表规则

- 只在前 30 行内定位关键字：
  - `种类`
  - `单价`
  - 可选：`昵称总数` / `昵称/总数` / `昵称`
- 找到后：
  - `label_col` 作为 CN 列
  - `label_col - 1` 作为总金额列
  - 角色列从 `label_col + 1` 向右展开
- 某列同时满足“角色名非空、单价大于 0”时，判定为有效角色列。
- 遇到“角色名为空且前面已经有角色列”时，停止继续向右扫描。
- 每个用户行里，只有数量大于 0 的角色项才展开为记录。
- 矩阵表生成的字段规则：
  - `item_name = group_name`
  - `batch = group_name`
  - `amount = quantity * unit_price`
  - `collected = false`

## `group_name` 推导

- 优先从左上角首格文本提取。
- 如果首格无法提取，则回退到工作表名。
- 再不行就回退到文件名去掉扩展名后的结果。

## 去重规则

- `record_id` 由以下字段拼接后做 MD5：
  - `source_file`
  - `source_sheet`
  - `batch`
  - `cn`
  - `item_name`
  - `role`
  - 同键重复出现时的序号

## 兼容字段别名

旧版兼容了多组中文表头别名，新的 Go 解析器至少要覆盖这些语义：

- `cn`：CN、买家、姓名、昵称、群昵称、下单人、收款对象
- `item_name`：谷子名称、谷名、商品名称、商品、周边、品名、款式、名称
- `role`：角色、人物、担、成员、种类
- `batch`：团购批次、批次、团名、车次、链接批次
- `quantity`：数量、件数、个数、qty
- `unit_price`：单价、价格、均价、price
- `amount`：金额、总金额、合计、小计、应收、实付、总价、尾款
- `collected`：是否已收、已收、收款状态、是否收款、收齐、已付款、paid

## 现阶段测试要求

- 两个真实源表必须继续解析成功。
- 非目标 Excel 不得误写入数据库。
- 新旧解析结果需要比对：
  - 总行数
  - 每个 `CN` 的金额汇总
  - 每个 `item_name + role` 的数量汇总
