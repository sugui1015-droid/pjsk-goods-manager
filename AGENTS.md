# AGENTS.md

## 开发日志与 Git 保护规则

- 每个实际修改批次（代码、配置、迁移、文档、页面或测试）完成时，必须同步更新开发日志。
- 开发日志统一保存在 `docs/development-logs/`。
- 同一天的小修改优先追加到当天汇总日志；较大的独立功能可新增专题日志，命名格式 `YYYY-MM-DD-英文简短功能名.md`。
- 禁止擅自删除、移动或重命名历史开发日志；如发现日志删除、重命名或移动，必须停止提交并向用户说明。
- 提交代码时必须同时提交对应日志；没有日志记录的实际修改批次不算完成。
- 不得通过 `.gitignore` 忽略 `docs/development-logs/`，日志需要由 Git 正常追踪和备份。
- 完成任务后必须汇报 Git 状态，包括未提交文件、暂存区状态，以及是否已提交或推送。

## 子代理使用限制

未经用户在当前任务中明确允许，禁止创建、启动、调用、委派或使用任何 subagent、子代理、并行代理、后台代理、worker 或 delegated agent。

不得以提高效率、并行调查、自动验收或后台检查等理由，自行把任务委派给其他代理执行。

该限制仅针对具有独立推理、独立任务执行或独立对话能力的代理，不限制当前主代理正常使用命令行、测试脚本、构建工具、浏览器工具、数据库工具或其他普通开发工具。

用户对某一次任务的子代理授权仅对该任务有效，不得自动延续到后续任务。

未获授权时，所有调查、修改、测试、验收和汇报必须由当前主代理直接完成。

任务完成后，必须在汇报中明确说明本轮是否使用了子代理。
